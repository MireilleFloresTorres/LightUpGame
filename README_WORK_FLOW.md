## Branches
- main: versión estable del proyecto.
- develop: integración de funcionalidades.
- feature/nombre-funcionalidad: desarrollo de nuevas características.
    - Ejemplo
        - feature/combat-system
        - feature/altar-system
- bugfix/nombre-error: corrección de errores.
  - Ejemplo
    - bugfix/Player-movement
    - bugfix/Enemy-controller
  
### Pull Requests
- Todos los cambios deberán realizarse mediante Pull Request.
- No se permitirán cambios directos en la rama main.

### Aprobación de Merges
- El Pull Request deberá ser aprobado por los integrantes de equipo.
- La integración final será realizada por el responsable del repositorio.

### Frecuencia de commits
- Realizar commits al finalizar una tarea especifíca.
- Realizar commits al finalizar cada sesión de trabajo.

## Convenciones de Nomenclatura
  ### Scripts
    - PascalCase
  - Ejemplos 
    - PlayerManager
    - GameManager

  ### Objetos
    -  Prefijo: obj_
  - Ejemplos:
    - obj_Player
    - obj_Altar
  ### Prefabs
    - Prefijo: pf_
  - Ejemplos:
    - pf_Player
    - pf_Altar
### Botones:
    - Prefijo: btn_
  - Ejemplos:
    - btn_Play
    - btn_Options
### Paneles
    - Prefijo: panel_
  - Ejemplos:
    - panel_MainMenu
    - panel_Pause
### Textos
    - Prefijo: txt_
  -  Ejemplos:
    - txt_Title
    - txt_Health
### Audio:
#### Musica:
    - Prefijo: bgm_
  - Ejemplos: 
    - bgm_MainMenu
    - bgm_Credits
#### Efectos:
      - Prefijo: sfx_
  - Ejemplos: 
    - sfx_Walk
    - sfx_Damage
### Escenas
    - Prefijo: scn_
- Ejemplos:
  - scn_MainMenu
  - scn_Credits

## Reglas de Trabajo
  - Evitar trabajar simultaneamente en la misma escena.
  - Mantener los prefabs organizados por carpetas.
  - No subir archivos temporales generados por Unity.
  - Actualizar la rama develop antes de comenzar a trabajar.
